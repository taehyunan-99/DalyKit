# Domain (도메인 파일 구조화)

> domain.md 상단 자유 입력 섹션을 읽고 → 구조화 섹션을 생성/갱신한다.

## 사용법

```
dalykit:domain     ← domain.md 자유 입력 + CSV 참조 → 구조화 섹션 갱신
```

## 사전 조건

- `dalykit/` 폴더가 존재해야 한다 (Glob으로 확인)
- 없으면: "`dalykit:init`을 먼저 실행하세요." 안내 후 종료
- `dalykit/config/domain.md`가 존재해야 한다
- 없으면: "`dalykit:init`을 먼저 실행하세요." 안내 후 종료

## domain.md 파일 구조

```markdown
# 자유 입력
<!-- 데이터/도메인에 대해 아는 것을 자유롭게 작성하세요. 형식 무관. -->
<!-- 예: "은행 고객 데이터, Personal Loan이 타겟, Experience 음수는 정상" -->


---

# 구조화 정보
<!-- dalykit:domain이 자동 생성. 직접 수정하지 마세요. -->

## 업종
...
```

## 워크플로우

### 1단계: 입력 수집

1. `dalykit/config/domain.md` Read → `# 자유 입력` 섹션 추출
2. `dalykit/data/` Glob → CSV/Excel 파일 탐색
  - 파일 존재 시: 컬럼명, dtypes, nunique, 기초 통계 파악 (head_limit 적용)
  - 1000행 이상 시 `.sample(1000)` 기준으로 처리

### 2단계: 구조화

자유 입력 + CSV 정보를 종합하여 아래 항목을 채운다:

| 항목 | 소스 우선순위 |
|------|--------------|
| 업종 | 자유 입력 → CSV 컬럼명 추론 |
| 데이터 설명 | 자유 입력 |
| 타겟 변수 | 자유 입력 → 이진/범주형 컬럼 추론 |
| 주요 피처 | 자유 입력 → 수치형 컬럼 자동 나열 |
| 컬럼 설명 | 자유 입력 → 컬럼명 기반 추론 (단위·제어 가능 여부 포함) |
| 도메인 규칙 | 자유 입력 (추론 불가 — 사용자 입력 우선) |
| 비즈니스 제약 | 자유 입력 (ID/개인정보 컬럼은 자동 탐지 후 제안) |
| 분석 제외 컬럼 | 자유 입력 → ID성 컬럼(nunique ≈ 행 수) 자동 탐지 |

### 3단계: domain.md 갱신

- `# 자유 입력` 섹션: **그대로 유지**
- `---` 구분선 이하 `# 구조화 정보` 섹션: **덮어쓰기**
- Edit 도구로 `# 구조화 정보` 섹션만 교체

### 4단계: 완료 안내

```
domain.md 구조화 완료.
자유 입력은 그대로 유지되었습니다.
구조화 정보를 검토한 뒤 필요한 항목을 직접 수정하세요.
(특히 도메인 규칙, 비즈니스 제약은 AI 추론 한계가 있습니다)
```

## 구조화 정보 출력 형식

```markdown
# 구조화 정보
<!-- dalykit:domain이 자동 생성. 직접 수정하지 마세요. -->

## 업종
- 업종: {값}

## 데이터 설명
- 설명: {값}

## 핵심 타겟 변수
- 타겟: {값}

## 주요 피처
- 피처: {값}

## 컬럼 설명
| 컬럼명 | 설명 | 단위 | 제어 가능 여부 |
|--------|------|------|--------------|
| ...    | ...  | ...  | ...          |

## 도메인 규칙
- {자유 입력에서 추출한 규칙 / 없으면 "미지정 — domain.md에 직접 추가하세요"}

## 비즈니스 제약
- {자유 입력에서 추출 / ID성 컬럼 자동 탐지 결과}

## 분석 제외 컬럼
- {ID성 컬럼 목록 (nunique ≈ 행 수 기준)}

## 특이사항
- {자유 입력에서 추출 / 없으면 생략}
```

## 규칙

1. **자유 입력 보존**: `# 자유 입력` 섹션은 절대 수정하지 않는다
2. **추론 명시**: AI가 추론한 항목은 `(추론)` 태그 표시
3. **불확실 항목**: 추론 불가 시 "미지정 — 직접 입력하세요" 표기
4. **한국어**: 모든 설명은 한국어로 작성
