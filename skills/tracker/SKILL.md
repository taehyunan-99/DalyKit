# Tracker (작업 추적 업데이트)

`docs/work-tracker.md`를 현재 상태에 맞게 자동 갱신한다.

> 개발 전용 스킬 — 플러그인 배포 시 제외.

## 사용법

```
harnessda:tracker              ← 전체 자동 업데이트
harnessda:tracker [변경 설명]  ← 설명 지정하여 업데이트
```

## 워크플로우

### 1단계: 변경사항 수집

1. **git diff**: `git diff --name-only HEAD~1` 또는 `git status --porcelain`으로 변경 파일 목록 수집
2. **git log**: `git log --oneline -5`로 최근 커밋 메시지 확인
3. **스킬/에이전트 목록 스캔**: `skills/*/SKILL.md`, `agents/*.md` Glob으로 현재 존재하는 항목 파악

### 2단계: work-tracker.md 갱신

Read로 `docs/work-tracker.md`를 읽고 아래 섹션을 Edit로 업데이트한다.

#### 갱신 대상 섹션

| 섹션 | 갱신 규칙 |
|------|----------|
| `마지막 업데이트` | 오늘 날짜로 변경 |
| `현재 상태` | 1줄 요약 — 직전 작업 + 다음 단계 |
| `최근 변경` | 새 행 추가 (LIFO, 최대 5개 유지 — 초과 시 가장 오래된 항목 삭제) |
| `진행 중` | 현재 세션에서 시작했으나 미완료된 작업 |
| `대기열` | 완료된 항목에 ~~취소선~~ ✅ 추가 |

#### 갱신하지 않는 섹션

| 섹션 | 이유 |
|------|------|
| `보류/제외` | 사용자 판단 필요 — 자동 수정 금지 |
| `설계 결정 로그` | 사용자 인자로 전달된 경우에만 추가 |

### 3단계: 설계 결정 추가 (선택)

사용자가 인자로 설계 결정을 전달하면 `설계 결정 로그`에 새 행 추가:

```
harnessda:tracker "da.md 제거 → harnessda:help 스킬로 대체 | 플러그인 네임스페이스 통일"
```

→ 설계 결정 로그에 추가:
```
| 2026-03-29 | da.md 제거 → harnessda:help 스킬로 대체 | 플러그인 네임스페이스 통일 |
```

인자에 `|`가 포함되면 `결정 | 근거` 형식으로 파싱한다.

## 최근 변경 행 형식

```markdown
| {오늘 날짜} | {변경 요약} | {영향 파일 목록} |
```

- **변경 요약**: git 커밋 메시지 또는 사용자 인자 기반
- **영향 파일**: `git diff --name-only`에서 추출, 프로젝트 루트 기준 상대경로

## 규칙

1. **Edit 도구만 사용** — work-tracker.md 전체를 Write로 덮어쓰지 않는다
2. **최근 변경 5개 제한** — LIFO, 오래된 항목은 삭제
3. **보류/제외 섹션 수정 금지** — 사용자 판단 영역
4. **설계 결정은 명시적 요청 시에만** — 자동 추가 금지
5. **커밋 없음** — 트래커 업데이트만 하고 git commit은 하지 않는다
