# 통계 분석 보고서 생성 가이드

## 트리거

`harnessda:stat report` 호출 시 실행

## 결과 파싱 방법

1. Read로 `stat_results.json` 읽기
2. `output['summary']` → 요약 테이블
3. `output['results'][i]['assumptions']` → 가정 검정 결과
4. `output['results'][i]['posthoc']` → 사후 분석 결과

## 저장 규칙

- 경로: 프로젝트의 `docs/stat_report.md`
- `docs/` 없으면 Bash `mkdir -p docs` 실행
- Write 도구로 저장 (덮어쓰기 허용)

## 보고서 구조

### 1. 연구 개요

- 분석 목적 및 데이터 설명
- 검정 선택 근거
- **대표본 주의** (n > 5000일 경우 상단에 추가):

> 본 데이터는 대표본(n={n:,})으로 검정력이 높아 작은 차이도 유의하게 검출됩니다.
> p-value와 효과 크기를 반드시 병행 해석하십시오.

### 2. 전제 조건 확인

- 정규성 검정 결과 (Shapiro-Wilk / Jarque-Bera)
- 등분산성 검정 결과 (Levene)

### 3. 가설 검정 결과 (핵심 — 귀무가설 초점)

각 검정마다 아래 형식으로 작성. **H0 기각/기각 불가가 반드시 첫 문장**:

```markdown
#### 검정 1: [분석 제목]

**H0**: [귀무가설 내용]
**결론**: H0 기각 / H0 기각할 수 없음 (p=0.xxx, α=0.05)

[기각 시]
[검정명] 결과, [X변수]에 따른 [Y변수]에 유의한 차이가 있다.
효과 크기 [η²=0.xxx → 소/중/대규모 효과].
[사후 분석] 유의한 그룹 쌍: [A vs B (p=xxx), ...]

[기각할 수 없음 시]
[검정명] 결과, [X변수]에 따른 [Y변수]에 유의한 차이가 있다고 볼 수 없다 (p=0.xxx).
```

## 효과 크기 해석 기준

| 유형 | 매우 작음 | 작음 | 중간 | 큼 |
|------|----------|------|------|-----|
| Cohen's d (2그룹) | <0.2 | 0.2~0.5 | 0.5~0.8 | ≥0.8 |
| η² (다그룹) | <0.01 | 0.01~0.06 | 0.06~0.14 | ≥0.14 |
| r (상관) | <0.1 | 0.1~0.3 | 0.3~0.5 | ≥0.5 |
| Cramér's V | <0.1 | 0.1~0.3 | 0.3~0.5 | ≥0.5 |

## 해석 문장 템플릿

**기각 + 큰 효과**:
> [검정명] 결과 유의한 차이가 있으며(p=xxx), 효과 크기도 큼(η²=xxx). 실질적으로 의미 있는 차이다.

**기각 + 작은 효과 + 대표본**:
> [검정명] 결과 통계적으로 유의하나(p=xxx), 효과 크기가 매우 작아(η²=xxx) 대표본에 의한 검정력 과잉일 수 있다.

**기각할 수 없음**:
> [검정명] 결과 유의한 차이가 있다고 볼 수 없다(p=xxx). [X변수]가 [Y변수]에 유의한 영향을 미친다는 증거가 부족하다.
